La Arquitectura Cliente-Servidor

La estructura cliente-servidor es básicamente la espina dorsal de todo internet. En la práctica es tan básico como si estuvieses preguntando por la hora a alguien, pero en internet es un poco más privado que esa analogía.

Los Actores

El Cliente

Es aquel que hace peticiones al servidor. Puede ser un navegador tal como Firefox, Google Chrome o Safari; una aplicación en tu computadora o un juego online.

El Servidor

Es una máquina de gran capacidad que siempre está encendida, esperando solicitudes. Su rol es gestionar recursos y enviar una respuesta válida.

¿Cuál es el ciclo de esta estructura?

  1. Petición (Request)

    2. El cliente envía un mensaje solicitando un servicio o recurso.

  2. Procesamiento

    3. El servidor recibe la petición, valida la identidad y procesa los datos.

  3. Respuesta (Response)

    4. El servidor envía el resultado. Si falla, recibes errores como el clásico 404.

Ventajas y Desventajas

Ventajas

Facilita la actualización de datos y permite que dispositivos de poca potencia accedan a procesos complejos.

Desventajas

Si el servidor cae, el servicio se interrumpe para todos. Puede haber congestión por exceso de tráfico.
